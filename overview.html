mental_health_data — Package Overview

This package automates the collection and transformation of Oregon mental health system data into structured CSV files used as inputs to a Simio discrete-event simulation model of the Aid & Assist (A&A) forensic patient pipeline — from court order through Oregon State Hospital (OSH) admission, restoration, and discharge.

1. Data Sources & Downloads

All downloads are handled in build_model_data/download_data_updates.py. The script checks what files already exist locally and only fetches what is new.

This is more reliable and avoids unnecessary HTTP requests.

2. PDF Parsing & Aggregation

All processing is in build_model_data/process_a_a_data.py. Each function reads every PDF in its directory, extracts tables using tabula-py (requires Java), validates the data, and stacks it into a long-format CSV timeseries.

A. A&A Census by County — process_a_a_census_timeseries()

Reads weekly census PDFs. Each report is a single table showing, for every Oregon county:

  • A&A Census — total defendants on the Aid & Assist waitlist
  • .370 / .365 / .315 Census — counts by ORS statute type (felony severity)
  • Fel. / Misd. — felony and misdemeanor counts and percentages
  • None Listed — defendants with no charge type listed
  • Change from Prev. Week — week-over-week delta
  • % of State Pop. / Census vs. Pop. Dif. — population-adjusted metrics

The table is validated (column totals must match the sum of county rows, within a rounding tolerance of ±1), then pivoted to long format with columns: County, Date, Variable, Value.

Output: OSH AandA Census/osh_a_a_census_timeseries_through_YYYY-MM.csv

B. Forensic Admission & Discharge Dashboard — process_aa_admit_discharge_timeseries()

Reads monthly forensic dashboard PDFs. The relevant page is identified by scanning for text containing "Aid & Assist", "Baker", and "Low High Count". Each report contains one wide table with 4 logical groups of columns, each broken out into a separate long-format timeseries:

Output CSVColumns capturedAggregation check
osh_a_a_admission_list_through_YYYY-MM.csv Admission list count, avg days waiting, low, high Count = sum; Low = min; High = max across counties
osh_a_a_patients_admitted_through_YYYY-MM.csv Patients admitted count, avg days, low, high Same
osh_a_a_no_longer_needing_hloc_through_YYYY-MM.csv Patients no longer needing higher level of care — 8 LOCUS-scored columns Count = sum; avg/low/high validated
osh_a_a_patients_discharged_through_YYYY-MM.csv Patients discharged for community restoration — 8 LOCUS-scored columns Same

All outputs are long format: County, Date, Variable, Value.

C. Court Docket — update_docket_data()

Downloads the current Multnomah County court appearance list PDF and saves it with today's date. No parsing or aggregation — the raw PDF is the deliverable.

3. Data Validation

Before saving, every parsed table is validated:

  • Column totals: The "Total" row in each PDF must equal the sum of all county rows (±1 for rounding). If not, the script raises an AssertionError.
  • Table structure: The exact set of expected columns is checked via assert set(df.columns).issuperset({...}). Missing columns raise immediately.
  • Misaligned rows: tabula occasionally misaligns numeric values across columns due to whitespace in the PDF. The fix_misaligned_table() function uses combinatorial search (permutations) to find the correct column assignment that satisfies the known column totals.

5. Automation

build_model_data/update_data.bat is a Windows batch file that runs the full pipeline on a schedule (configured externally). It runs process_a_a_data.py as __main__, which sequentially calls all three active processing functions.
