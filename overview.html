mental_health_data — Package Overview

This package automates the collection and transformation of Oregon mental health system data into structured CSV files used as inputs to a Simio discrete-event simulation model of the Aid & Assist (A&A) forensic patient pipeline — from court order through Oregon State Hospital (OSH) admission, restoration, and discharge.

1. Data Sources & Downloads

All downloads are handled in build_model_data/download_data_updates.py. The script checks what files already exist locally and only fetches what is new.

This is more reliable and avoids unnecessary HTTP requests.

2. PDF Parsing & Aggregation

All processing is in build_model_data/process_a_a_data.py. Each function extracts tables using tabula-py (requires Java), validates the data, and stacks it into a long-format CSV timeseries.

On each run, both functions check for an existing CSV in the output directory. If one is found, they load it, identify the latest date already processed, and skip all PDFs on or before that date — only new PDFs are parsed and appended. The old dated CSV is replaced with a new one reflecting the updated date range.

A. A&A Census by County — process_a_a_census_timeseries()

Reads 182 weekly census PDFs (Sep 2022 – Apr 2026), covering 37 Oregon counties. Each report is a single table parsed with tabula-py and pivoted to long format.

Output: OSH AandA Census/osh_a_a_census_timeseries_through_YYYY-MM.csv — 87,542 rows, columns: County, Date, Variable, Value

B. Forensic Admission & Discharge Dashboard — process_aa_admit_discharge_timeseries()

Reads 36 monthly forensic dashboard PDFs (Apr 2023 – Mar 2026), covering 41 Oregon counties. The relevant page is identified by scanning PDF text for "Aid & Assist", "Baker", and "Low High Count". The wide table is split into 4 logical groups, each saved as a separate long-format CSV.

All four files share the same schema: County, Date, Variable, Value.

Output CSVRowsVariablesDescription
osh_a_a_admission_list_through_YYYY-MM.csv 5,328 Count, Avg Days, Low, High Defendants currently on the OSH admission waitlist by county. Count = number waiting; Avg Days = average wait time; Low/High = min/max individual wait times that month.
osh_a_a_patients_admitted_through_YYYY-MM.csv 5,328 Count, Avg Days, Low, High Defendants admitted to OSH that month by county. Same four variables as admission list — count of admissions, average days from order to admission, shortest and longest waits.
osh_a_a_no_longer_needing_hloc_through_YYYY-MM.csv 10,656 Count, Avg Days, LOCUS_1 – LOCUS_6 Defendants who were removed from the OSH waitlist because they no longer required a higher level of care (HLOC). LOCUS scores (1–6) indicate clinical acuity at the time of removal, where higher scores reflect greater impairment.
osh_a_a_patients_discharged_through_YYYY-MM.csv 10,656 Count, Avg Days, LOCUS_1 – LOCUS_6 Defendants discharged from OSH for community restoration that month. LOCUS scores (1–6) reflect clinical acuity at discharge. Avg Days = average length of OSH stay.

Validation: Count columns are verified to sum correctly across counties; Low = county minimum, High = county maximum in the statewide total row.

C. Court Docket — update_docket_data()

Downloads the current Multnomah County court appearance list PDF from mcda.us and saves it with today's date. No parsing or aggregation — the raw PDF is the deliverable, used to track upcoming A&A court dates.

3. Data Validation

Before saving, every parsed table is validated:

  • Column totals: The "Total" row in each PDF must equal the sum of all county rows (±1 for rounding). If not, the script raises an AssertionError.
  • Table structure: The exact set of expected columns is checked via assert set(df.columns).issuperset({...}). Missing columns raise immediately.
  • Misaligned rows: tabula occasionally misaligns numeric values across columns due to whitespace in the PDF. The fix_misaligned_table() function uses combinatorial search (permutations) to find the correct column assignment that satisfies the known column totals.

5. Automation

The pipeline is deployed on an Ubuntu 22.04 LTS server (nash2) and scheduled via cron. Output CSVs are written to a Samba share accessible from other machines on the network.

FilePurpose
build_model_data/update_data.sh Linux shell script — sets JAVA_HOME and PATH, then runs process_a_a_data.py with the Samba share paths as arguments. Logs to update_data_log.txt.
build_model_data/update_data.bat Legacy Windows batch file — retained for reference only.
